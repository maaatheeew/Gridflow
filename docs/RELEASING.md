# Releasing Gridflow

## What Gets Published

The GitHub release workflow publishes:

- GitHub's autogenerated source archives
- `Gridflow-<version>.dmg`
- `Gridflow-<version>.dmg.sha256`

## Release Checklist

1. Confirm the README, screenshots, and docs are current.
2. Make sure screenshots and sample content use demo data only.
3. Run a local build:

```bash
cd Gridflow
swift build --disable-sandbox --scratch-path "$PWD/.build/swiftpm" -c release
APP_VERSION=1.0.0 APP_BUILD_NUMBER=1 ./scripts/build_dmg.sh
```

4. Create an annotated tag such as `v1.0.0`.
5. Push the tag to GitHub.
6. Wait for the `Release` workflow to finish.
7. Verify that the release contains the DMG, checksum, and autogenerated source archives.
8. Smoke test the published DMG on a clean machine or clean user profile when possible.

## Notes

- The release workflow derives the DMG version from the Git tag.
- The repository ignores local exports and build artifacts so personal task data is not packaged into the release by accident.

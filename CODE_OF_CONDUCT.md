# Code Of Conduct

## Our Goal

Gridflow should be a welcoming project for contributors of all experience levels. We value clarity, kindness, curiosity, and respect.

## Expected Behavior

- Be respectful in issues, pull requests, reviews, and discussions.
- Assume good intent and ask clarifying questions before escalating.
- Give constructive feedback that focuses on the work, not the person.
- Share context that helps others reproduce, review, and improve changes.

## Unacceptable Behavior

- Harassment, discrimination, intimidation, or personal attacks
- Sexualized language or unwanted sexual attention
- Doxxing or publishing private information without consent
- Posting someone else's task data, screenshots, exports, or system details without permission
- Disruptive trolling, bad-faith arguing, or repeated boundary violations

## Scope

This Code of Conduct applies to the repository, issue tracker, pull requests, discussions, and any project spaces used to collaborate on Gridflow.

## Enforcement

If you experience or witness unacceptable behavior, contact the repository maintainer privately. Public call-outs are discouraged when a private report can resolve the issue more safely.

Maintainers may remove, edit, or reject comments, commits, pull requests, issues, and other contributions that violate this Code of Conduct.
